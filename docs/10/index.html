10 · 可复现分析与 R Markdown

1 为何要可复现

可复现分析指：他人（或未来的自己）能用同一份代码和数据得到相同结果。医学研究与论文审稿越来越强调“数据与代码可得、分析可复现”。用脚本而非点击操作、用 R Markdown 把说明、代码和结果写在一起、固定随机种子与包版本，都是可复现的基础。

2 R Markdown 基本结构

R Markdown（.Rmd）文件由三部分构成：顶部的 YAML 头（标题、作者、输出格式等）、正文（Markdown 语法）、以及嵌入的 R 代码块。knit 后生成 HTML/PDF/Word 报告，代码与结果自动嵌入。

2.1 YAML 与标题

# 在 .Rmd 文件顶部（三个短横线之间），例如：
---
title: "数据分析报告"
author: "姓名"
date: "`r Sys.Date()`"
output: html_document
---

2.2 正文与代码块

正文用 # ## ### 写标题，普通段落直接写。R 代码放在三个反引号 + {r} 的代码块里，knit 时会执行并输出结果与图形。

3 代码块与输出

# 代码块示例（在 .Rmd 中写作）：
```{r load-data, echo=TRUE, message=FALSE}
library(dplyr)
d <- read.csv("data.csv")
summary(d)
```

# 选项：echo=TRUE 显示代码；eval=FALSE 只显示不运行；
# fig.width=6 控制图宽；warning=FALSE 隐藏警告。

合理使用代码块选项（如 echo、include、fig.cap）可让报告更清晰；关键分析建议保留代码以便审稿与复现。

4 小结与练习

本节要求：理解可复现分析的意义；能新建 .Rmd、写 YAML 与标题、插入 R 代码块并 knit 出 HTML 报告；知道常用代码块选项。

要点：用 .Rmd 整合文字与代码、定期 knit 检查、配合版本控制（如 Git）与固定包版本，是建立可复现流程的实用做法。
