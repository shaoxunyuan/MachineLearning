02 · 数据描述性统计

一、医学数据基础

医学数据是进行数据分析与挖掘的起点。理解医学数据的类型、特征与来源，有助于制定合理的数据质量策略与分析方法。

1.1 医学数据类型与特点

  • 结构化数据：电子病历、检验指标、用药记录
  • 非结构化数据：病历文本、医学影像、基因序列
  • 医学数据 5V 特征：海量、多样、高价值、实时、准确性
  • 医学数据特殊性：冗余、多属性、不完整、强时序性

1.2 医学数据来源

医学数据来源多样，可按用途与场景分类：

临床数据源
EHR/EMR、检验检查、手术记录、护理记录等
科研数据源
临床试验、队列研究、生物样本库等
公共数据源
公共卫生监测、医保数据、开放医学数据库等
新兴数据源
可穿戴设备、移动医疗、实时生理监测等

二、医学数据质量分析

数据质量分析是数据预处理的前提，主要任务是识别"脏数据"及其成因，并系统梳理数据质量问题的类型与识别方法。

2.1 脏数据及其原因和案例

"脏数据"指不符合数据质量要求的数据。常见成因与案例如下：

缺失数据
字段为空（NA、NULL、空字符串），如患者年龄未填写、检验结果缺失；原因包括未做检查、录入遗漏、系统未采集等。
异常数据
超出合理范围或违背医学常识，如血压 300 mmHg、年龄为负数；多由录入错误、单位混用或设备故障导致。
重复数据
同一条记录被多次录入，如同一患者同次就诊的检验记录重复；常来自多系统导出合并或重复提交。
不一致数据
格式、单位或编码不统一，如性别"男/女"与"1/2"混用、日期格式各异；多源整合时尤为常见。

2.2 数据质量问题识别

  • 缺失值：完全随机缺失（MCAR）、随机缺失（MAR）、非随机缺失（MNAR）；需结合缺失模式与业务判断机制
  • 噪声与异常值：测量误差、录入错误、真实极端值；可通过 3σ、IQR、箱线图及领域规则识别
  • 重复数据：完全重复行与部分重复（同主键多记录）；需定义主键与去重/合并策略
  • 不一致性：编码不统一、单位不一致、逻辑矛盾（如出院早于入院）；需制定统一标准与一致性检查

三、医学数据清洗

数据清洗是数据预处理的核心步骤，包括对单表/单源数据的清洗与多源数据的集成、转换及特征工程。

3.1 数据清洗

围绕完整性、准确性、一致性与可分析性，通常开展以下四类处理：

缺失值处理：删除、均值/中位数/众数填充、KNN 插值、模型预测（如多重插补）。

异常值处理：3σ 原则、四分位距法、分箱平滑、专业领域规则剔除。

重复数据处理：去重、合并、冲突解决（如保留最新或非缺失优先）。

数据标准化：单位统一、编码标准化（如 ICD-10、SNOMED-CT）。

缺失值处理示例

# 假设 df 为包含缺失值的数据框

# 1) 删除（整行删除）
df_drop <- na.omit(df)

# 2) 简单填充（示例：年龄用中位数；分类变量可用众数）
df$age[is.na(df$age)] <- median(df$age, na.rm = TRUE)

# 3) KNN 插补（VIM 包）
library(VIM)
df_knn <- kNN(df, k = 5)  # 默认会产生 *_imp 标记列

# 4) 模型预测/多重插补（mice 包：pmm）
library(mice)
imp <- mice(df, m = 5, method = "pmm", seed = 123)
df_model <- complete(imp, 1)

异常值处理示例

# 假设 x 为某个连续指标向量

# 1) 3σ 原则
mu <- mean(x, na.rm = TRUE)
s <- sd(x, na.rm = TRUE)
is_out_3s <- (x < (mu - 3 * s)) | (x > (mu + 3 * s))

# 2) IQR 法（箱线图规则）
q1 <- quantile(x, 0.25, na.rm = TRUE)
q3 <- quantile(x, 0.75, na.rm = TRUE)
iqr <- q3 - q1
is_out_iqr <- (x < (q1 - 1.5 * iqr)) | (x > (q3 + 1.5 * iqr))

# 3) 分箱平滑（用每个箱的中位数替代）
bins <- cut(x, breaks = quantile(x, probs = seq(0, 1, 0.1), na.rm = TRUE), include.lowest = TRUE)
x_smooth <- ave(x, bins, FUN = function(v) median(v, na.rm = TRUE))

# 4) 领域规则示例：将不可能的年龄标记为 NA（如 <0 或 >120）
# df$age[df$age < 0 | df$age > 120] <- NA

重复数据处理示例

library(dplyr)

# 1) 完全去重
df1 <- df %>% distinct()

# 2) 按主键去重：保留最新一条（示例：patient_id + visit_date）
df2 <- df %>%
  arrange(patient_id, visit_date, desc(update_time)) %>%
  distinct(patient_id, visit_date, .keep_all = TRUE)

# 3) 冲突合并：同主键下，对数值取平均；对字符取非缺失优先
df3 <- df %>%
  group_by(patient_id, visit_date) %>%
  summarise(
    sbp = mean(sbp, na.rm = TRUE),
    diagnosis = dplyr::first(na.omit(diagnosis)),
    .groups = "drop"
  )

数据标准化示例

library(dplyr)

# 1) 单位统一示例：血糖 mg/dL -> mmol/L（近似：除以 18）
df$glu_mmol_l <- ifelse(df$glu_unit == "mg/dL", df$glu_value / 18, df$glu_value)
df$glu_unit <- "mmol/L"

# 2) 编码标准化示例：性别字段统一
df$sex_std <- dplyr::case_when(
  df$sex %in% c("男", "M", "1") ~ "男",
  df$sex %in% c("女", "F", "2") ~ "女",
  TRUE ~ NA_character_
)

3.2 数据集成与转换

  • 多源数据融合：关键字匹配、规则匹配、机器学习匹配（实体对齐、记录链接）
  • 数据转换：标准化（Z-score）、归一化（Min-Max）、对数变换、离散化
  • 特征工程：特征提取、特征选择、特征构造、降维（PCA/FA）

四、挑战与伦理

医学数据分析与数据挖掘在带来价值的同时，也面临技术难点与伦理、法规约束。本节从技术挑战与伦理法规两方面简要归纳。

4.1 技术挑战

  • 数据质量：缺失、噪声、异构性（多源、多格式、多标准），影响模型稳健性与结论可靠性
  • 隐私保护：患者敏感信息安全、数据匿名化与脱敏，在可用性与隐私之间取得平衡
  • 可解释性：黑箱模型（如复杂深度学习）带来的临床信任问题，需可解释 AI 与临床可接受性
  • 标准化：多中心数据整合、术语与编码统一（如 ICD、SNOMED），实现跨机构可比与复用

4.2 伦理与法规

  • 知情同意：数据使用授权、研究目的告知、隐私保护承诺，确保患者知情与同意
  • 数据安全：存储加密、传输安全、访问控制与权限管理，防止泄露与滥用
  • 法规遵循：HIPAA（美国）、GDPR（欧盟）及国内医疗数据管理规范，合规使用与跨境注意适用法律
