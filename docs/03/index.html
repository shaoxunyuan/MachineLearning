03 · 数据清洗

1 清洗流程概述

数据清洗是在质量评估之后、建模分析之前的关键环节：在尽量不引入新偏差的前提下，把缺失、重复、异常、类型错误等问题处理成“可分析”的整洁数据。医学研究中清洗决策（如删缺失还是填补、异常值删还是截尾）往往需要结合专业判断并记录在分析报告中。

1.1 目标与原则

目标：得到每一行对应一条有效观测、每一列类型正确且含义明确的数据框。原则：能追溯（保留原始数据或清洗脚本）、可复现（用代码而非手工改表）、有依据（删除或填补要有理由并说明）。

1.2 典型步骤

常见顺序为：缺失值处理（删除或填补）→ 重复行识别与去重 → 异常值识别与处理（删除/截尾/保留并标注）→ 类型与格式统一（数值/因子/日期等）。具体顺序可根据数据情况调整，但建议先缺失与重复，再异常与类型。

2 缺失值处理

缺失要么导致后续函数报错或结果变为 NA，要么在建模时被默认删除，影响样本量。处理方式主要有“删除”和“填补”两类；选择哪种需结合缺失机制与分析目的。

2.1 删除与保留

若缺失很少且可认为“完全随机”，可对分析用到的列做“完整个案”筛选；若某列缺失过多且非关键，可先删列再分析。注意：行删除会改变样本量，需在报告中说明。

# 只保留指定列全部非缺失的行（完整个案）
d_comp <- d[complete.cases(d[, c("age", "dose_g", "zheng")]), ]

# 或使用 dplyr：按条件筛选
library(dplyr)
d_comp <- d %>% filter(!is.na(age), !is.na(dose_g))

2.2 简单填补

当缺失不可忽略且不宜删行时，可考虑简单填补：数值列常用均值、中位数或分组中位数；分类列可用众数或“缺失”作为单独类别。复杂方法（如多重插补）在后续课程或专题中再学。

# 用全列中位数填补 dose_g 的缺失（示例）
med <- median(d$dose_g, na.rm = TRUE)
d$dose_g_filled <- ifelse(is.na(d$dose_g), med, d$dose_g)

# 按证型分组用组内中位数填补（更合理）
d <- d %>% group_by(zheng) %>%
  mutate(dose_g_filled = ifelse(is.na(dose_g), median(dose_g, na.rm = TRUE), dose_g)) %>%
  ungroup()

3 重复与异常值

重复行可能来自重复录入或多次导出；异常值可能是录入错误或真实极端值。两者都需要先识别再决定处理方式。

3.1 重复行

# 检测重复行（所有列都相同）
duplicated(d)
sum(duplicated(d))

# 去重：保留第一次出现的行
d_uniq <- d[!duplicated(d), ]

# 按“患者ID + 就诊日期”判定重复，保留第一条
d_uniq <- d %>% distinct(patient_id, visit_date, .keep_all = TRUE)

3.2 异常值处理思路

异常值处理没有统一答案：可删除、截尾（如将超过 P99 的改为 P99）、或保留并做敏感性分析。医学上常结合临床合理范围（如收缩压 80–200 mmHg）做判断。

# 示例：将 dose_g 中大于 100 的视为异常并改为 NA（或截尾为 100）
d$dose_g_adj <- ifelse(d$dose_g > 100, NA, d$dose_g)

# 或按分位数截尾
q99 <- quantile(d$dose_g, 0.99, na.rm = TRUE)
d$dose_g_adj <- pmin(d$dose_g, q99)

4 类型与格式统一

读入的列有时是字符型（如“12”“男/女”），需要转为数值或因子；日期可能以字符串形式存在，需用 as.Date 或 lubridate 统一。统一后便于后续运算与建模。

# 字符转数值（无法转换的会变 NA）
d$age <- as.numeric(d$age)

# 字符转因子并指定水平顺序
d$sex <- factor(d$sex, levels = c("女", "男"))
d$zheng <- factor(d$zheng, levels = c("气虚", "血虚", "湿热", "痰湿"))

5 小结与练习

本节要求：能根据评估结果对缺失做删除或简单填补、对重复行去重、对异常值做合理处理，并能把字符/日期等统一为合适类型。所有操作建议在脚本中完成并注释理由。

要点：清洗要有据可查、可复现；complete.cases、duplicated、distinct、mutate 与 as.numeric / factor 是常用工具。
