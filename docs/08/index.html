08 · 关联规则算法

一、关联规则与“购物篮分析”

关联规则（Association Rules）最经典的应用是购物篮分析：一位顾客在同一次交易中买了哪些商品，这些商品之间是否存在“经常一起出现”的模式。

在医学数据中，可以类比为：

  • 同一次门诊/住院记录中开具了哪些药物（处方组合）。
  • 同一个病人在一次就诊中做了哪些检查项目（检查套餐）。
  • 某些病理诊断结果是否总是与特定影像学提示同时出现。
目标：从大量“事务”（transaction）中，自动找出形如 {A,B} ⇒ {C} 的规则，并用支持度、置信度、提升度等指标衡量其“可信度”和“有用程度”。
本节部分思路参考了知乎专栏与掘金的入门文章（如 Apriori 与 FP-Growth 介绍、FP-Growth 实战）。

1.1 问题背景与经典示例

购物篮示例

每一行是一位顾客的一次购物记录：

  • T1: {牛奶, 面包, 鸡蛋}
  • T2: {啤酒, 尿布, 牛奶}
  • T3: {牛奶, 尿布, 面包, 啤酒}

我们关心：买了 {尿布} 的顾客，是否经常同时买 {啤酒}？

医学类比

每一行是一位病人的一次就诊：

  • V1: {头孢, 布洛芬, 血常规}
  • V2: {二甲双胍, 胰岛素, 空腹血糖}
  • V3: {ACEI, 利尿剂, 肌酐, 钾离子}

我们关心：哪些药物/检查组合经常同时出现，是否提示潜在的“处方模式”或“临床路径”。

1.2 支持度、置信度与提升度

  • 支持度（support）：同时包含 X 和 Y 的事务占全部事务的比例，衡量“规则出现的频繁程度”。
  • 置信度（confidence）：在包含 X 的事务中，同时也包含 Y 的比例，表示“在 X 发生时 Y 出现的条件概率”。
  • 提升度（lift）：\(lift(X⇒Y) = \frac{support(X∪Y)}{support(X)\times support(Y)}\)，衡量 X 与 Y 相比独立情形下的“关联强度”。
经验规则：在实际分析中，先给定一个较低的 minsup（最小支持度）筛掉极少出现的组合，再用 confidence 与 lift 筛选出真正有价值的规则。

二、Apriori 算法

Apriori 是最经典的关联规则挖掘算法，其核心在于反单调性（Apriori 性质）：

反单调性：若一个项集是频繁的，则它的所有子集也必须是频繁的；反之若某个项集是不频繁的，那么包含它的所有超集都一定不频繁，可直接剪枝。

2.1 核心思想：由小到大扩展

  1. 先遍历一次数据，统计所有 1-项集的支持度，得到频繁 1-项集 \(L_1\)。
  2. 用 \(L_1\) “自连接”生成候选 2-项集 \(C_2\)，剪掉任何子集不在 \(L_1\) 中的组合，得到频繁 2-项集 \(L_2\)。
  3. 重复上述过程：由 \(L_k\) 生成候选 \(C_{k+1}\)，计算支持度并剪枝，获得 \(L_{k+1}\)。
  4. 当 \(L_k\) 为空时停止，合并所有非空 \(L_k\) 得到全部频繁项集，再据此生成关联规则。
优点
  • 思想简单，便于讲解和实现。
  • 剪枝策略清晰，容易调试。
缺点
  • 需要多次扫描原始数据库。
  • 候选项集数量可能爆炸，内存/时间开销较大。

2.2 R 中使用 arules::apriori

在 R 中，可以利用 arules 包直接完成 Apriori 挖掘。下面用经典的 Groceries 事务型数据集演示。

options(repos = c(CRAN = "https://mirrors.tuna.tsinghua.edu.cn/CRAN/"))
install.packages("arules")  # 若已安装会提示跳过

library(arules)
data("Groceries")

Groceries
summary(Groceries)  # 查看事务数与项的概况

# 使用 Apriori 发现频繁模式与规则
rules <- apriori(
  Groceries,
  parameter = list(
    supp = 0.01,     # 最小支持度（至少 1% 的事务中出现）
    conf = 0.5,      # 最小置信度
    minlen = 2       # 至少包含 2 个项
  )
)

rules

# 按提升度从高到低排序，并查看前若干条
rules_sorted <- sort(rules, by = "lift", decreasing = TRUE)
inspect(head(rules_sorted, 10))

实际工作中，可以根据样本量调整 supp 与 conf，并用 subset() 函数筛选出“感兴趣的前件或后件”，例如只看“涉及某个关键药物”的规则。

三、FP-Growth 算法

FP-Growth（Frequent Pattern Growth）是为了解决 Apriori 中“候选项集过多、需要反复扫描数据库”的性能瓶颈而提出的改进算法。

3.1 FP-Tree 思路

  1. 首先扫描数据库，得到各个项的支持度，并按频繁程度排序。
  2. 再次扫描数据库，把每条事务中出现的频繁项按上述顺序插入到一棵压缩的前缀树（FP-Tree）中，相同前缀共享路径。
  3. 在 FP-Tree 上进行递归式“模式基”挖掘，不再显式生成大量候选项集。
直观理解：FP-Tree 把大量重复的事务前缀压缩在一起，相当于把“条条流水账”整理成了一棵紧凑的树，再在树上做递归搜索，避免了 Apriori 中庞大的候选枚举。

3.2 与 Apriori 的比较

Apriori
  • 显式生成大量候选项集。
  • 需要多次完整扫描数据库。
  • 实现简单、易于教学。
FP-Growth
  • 通过 FP-Tree 压缩数据，不显式枚举所有候选。
  • 通常只需两次扫描数据库，效率更高。
  • 实现相对复杂，但在大规模数据上更实用。

在许多工程实践（包括部分开源实现与大数据平台）中，FP-Growth 已经成为挖掘频繁项集与关联规则的主力算法之一，相关思想在诸多教程与实践文章中都有详细介绍（例如掘金上的 FP-Growth 实战文章）。

四、医学场景示例：处方/检查组合

下面给出一个模拟的处方数据示例，演示如何在 R 中将“长表格式”的就诊记录转成事务数据，再用 Apriori 进行挖掘。真实项目中，你只需把示例数据框替换为自己的 EMR/处方数据即可。

4.1 数据整理为“事务”

library(arules)

# 假设有一个长表：每行是一条“病人-就诊-药物”记录
presc_long <- data.frame(
  visit_id = c(1,1,1, 2,2, 3,3,3,3, 4,4),
  drug     = c("头孢", "布洛芬", "蒙脱石散",
               "二甲双胍", "胰岛素",
               "ACEI", "利尿剂", "他汀", "阿司匹林",
               "二甲双胍", "他汀"),
  stringsAsFactors = FALSE
)

presc_long

# 按 visit_id 聚合为“事务”
presc_split <- split(presc_long$drug, presc_long$visit_id)
presc_trans <- as(presc_split, "transactions")

presc_trans
inspect(presc_trans)

4.2 规则挖掘与解读

# 挖掘处方组合中的关联规则
rules_rx <- apriori(
  presc_trans,
  parameter = list(
    supp = 0.2,    # 由于示例事务很少，支持度可以设得高一些
    conf = 0.5,
    minlen = 2
  )
)

rules_rx
inspect(sort(rules_rx, by = "lift"))

在真实的处方/检查数据中，可以进一步：

  • 限制前件/后件的内容，例如只看“前件包含某诊断”或“后件包含某检查”的规则。
  • 结合临床知识过滤掉明显“无意义”的组合（例如所有人都会做的常规检查）。
  • 与不良事件、再入院等结局结合，挖掘潜在高风险组合。

五、小结与练习

本章介绍了关联规则挖掘的三大核心指标（支持度、置信度、提升度），以及两种经典算法：Apriori 与 FP-Growth。Apriori 便于理解与教学，而 FP-Growth 更适合大规模数据场景。

练习建议：
  • 使用 Groceries 数据，尝试不同 supp/conf 参数组合，观察规则数量与质量如何变化。
  • 构造一个“检查项目”模拟数据集，包含 50–100 条“就诊-检查”记录，并用 Apriori 挖掘规则。
  • 阅读一些工程实现（如 Spark MLlib 等）中对 FP-Growth 的介绍，思考在大数据平台上如何部署关联规则挖掘流程。
